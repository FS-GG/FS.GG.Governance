Credible evidence is **public-surface** testing only: `Ship.rollup` exercised through the built library over **real typed values** — real F018 `Gate`s, F017 `UnknownGovernedPathFinding`s, F019 `RouteResult`s, and the F023 `RunMode`/`Profile` levers — never private helpers and never mocks (Principle V). The hidden gate/finding→`EnforcementInput` mappings and the sort-key helper are tested **only through** `rollup`. Every input is a real, enumerable typed value — **no synthetic evidence is anticipated** (the per-item input domain is the finite cross-product of gate maturity / finding zone × run mode × profile, all literally constructible); if any `Synthetic`-tokened test ever appears it carries a use-site `// SYNTHETIC:` disclosure and is listed in the PR.

**Tier**: the whole feature is **Tier 1** (plan Constitution Check). Every task matches the feature tier; no per-task `[T1]`/`[T2]` annotations needed. **No existing project's public surface is touched** — `FS.GG.Governance.Enforcement` (F023) and `FS.GG.Governance.Route` (F019) are referenced as-is; the only new baseline is `surface/FS.GG.Governance.Ship.surface.txt`.

**Elmish/MVU (Principle IV)**: **NOT APPLICABLE** — this feature is a pure, total, side-effect-free value-to-value computation (FR-008, FR-012): no I/O, no git sensing, no clock, no multi-step state, no retries, no effect. It is exactly the "single pure function" case Principle IV exempts from MVU ceremony (plan Constitution Check; the same call F015/F017/F018/F019/F021/F023 made). The boundary is the two pure modules `Ship.Model` (the result vocabulary) and `Ship.Ship` (`rollup` + hidden mappings/partition/sort) — no `Model`/`Msg`/`Effect`/`update`/interpreter.

**Determinism minimums (FR-009, SC-004)**: `rollup` reads only the `RouteResult` and the two typed levers — no clock, environment, ordering, or host-path value enters it. Each output list is sorted by a stable composite per-item key (`"gate:" + gateIdValue id` for a gate; `"finding:" + path + ":" + findingIdToken` for a finding — gates before findings), so the result never depends on input-arrival order. Consequence: identical inputs always yield a byte-identical `ShipDecision` — same verdict, same `Blockers`/`Warnings`/`Passing` (same ordering), same `ExitCodeBasis` (SC-004), proven by evaluating each input twice and comparing.

**Carry/exclusion minimums (FR-005, FR-006, FR-010, SC-003, SC-006, SC-007)**: every `EnforcedItem` carries the F023 `EnforcementDecision` **verbatim** — `BaseSeverity` byte-identical to the base severity the gate/finding mapping assigned (FR-006, the design's no-hide rule `docs/initial-design.md:575`/`:806`), plus maturity, mode, profile, effective severity, and reason (FR-005). The profile may relax/tighten effective enforcement but NEVER changes the reported base severity. The three item lists are a **disjoint, exhaustive** partition: `|Blockers| + |Warnings| + |Passing| = |SelectedGates| + |Findings.Findings|` (FR-010, SC-006) — no item dropped, none double-counted (F019 already union-deduped selected gates by `GateId` — research D7). The surface exposes **no** serialized-document/exit-code/cache/freshness/policy-dial member (FR-012, SC-007) — asserted by the surface baseline.

**Totality minimums (FR-002, FR-007, FR-008, FR-011, SC-001, SC-005)**: `rollup` pattern-matches only closed DUs (gate `Maturity`, finding `FindingZone`, the partition on `Severity`) exhaustively — **no wildcard** that hides a future case — maps over lists with total combinators, and does no partial function, parse, division, or I/O, so it cannot throw for any well-typed input, **including the empty `RouteResult`** (⇒ `Pass`, all three lists `[]`, `ExitCodeBasis = Clean`). `Verdict = Fail` iff at least one item derives effective `Blocking` (FR-002, SC-001); `ExitCodeBasis = Blocked` iff `Verdict = Fail`, else `Clean` (FR-007). A base-`Advisory` item is always a passing item — never escalated (FR-011).

**Scope-guard minimums (FR-012/FR-013)**: pure whole-change decision only — **no** `audit.json` (or other) serialized document, **no** process exit code (only the typed `ExitCodeBasis`), **no** cache-eligibility/freshness evaluation, **no** `policy.yml` parse or per-class dial, **no** I/O, **no** command, **no** CLI. The carried `RouteResult.Cost` is **not** evaluated or re-projected. The gate/finding→`EnforcementInput` mapping (FR-013) is the documented plan-time reconciliation (research D3/D4): a `block-on-*` gate ⇒ base `Blocking` / `observe`/`warn` ⇒ base `Advisory` (maturity verbatim); a `ProtectedBoundaryUnknown` finding ⇒ base `Blocking` + `block-on-ship` / a `GovernedRootUnknown` finding ⇒ base `Advisory` + `warn`. The library takes **two** inward references (`FS.GG.Governance.Enforcement`, `FS.GG.Governance.Route`) and adds **no** new third-party `PackageReference` (`System.*`/FSharp.Core only — the rollup needs no serialization).
